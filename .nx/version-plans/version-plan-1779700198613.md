fix(CartesianChart): split chart into `decorations` and `children` layers so the reveal animation only clips the data layer (axes, grid, scrubber, and reference lines stay fully visible). Replaces the SVG `ClipPath`-based reveal with an `Animated.View overflow:hidden` wrapper around the data SVG, which fixes an Android-only bug where animated clip paths were not re-evaluated by `react-native-svg`. Adds a new `decorations` prop on `<CartesianChart>` for the always-visible chrome layer; `children` becomes the animated data layer. `LineChart` automatically routes `<Line>` and `<Point>` into the data layer and everything else into `decorations`. Backwards compatible: when only `children` is provided to `<CartesianChart>` with no `decorations`, the legacy single-layer rendering is used (no reveal animation).
