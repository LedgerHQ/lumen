# DotSymbol

## Introduction

DotSymbol positions a small indicator at a configurable corner of a child element such as MediaImage or Spot. It supports two content modes: **image** (default) renders an image from a URL, and **icon** renders an SVG icon with a semantic background color.

## Anatomy

<Canvas of={DotSymbolStories.Base} />

- **Wrapper**: Relative container that preserves the child's layout
- **Child**: The primary element (e.g. MediaImage, Spot)
- **Dot**: Absolutely-positioned indicator with border and configurable shape

### Pin

Four corner placements are available: `bottom-end` (default), `top-end`, `bottom-start`, and `top-start`.

<Canvas of={DotSymbolStories.PinShowcase} />

### Shapes

<Canvas of={DotSymbolStories.ShapeShowcase} />

- **circle** (default): Fully rounded dot
- **square**: Rounded corners that scale with size

### Sizes

The dot size is driven by the parent MediaImage or Spot size via the mapping helpers `mediaImageDotSizeMap` and `spotDotSizeMap`.

<Canvas of={DotSymbolStories.SizeShowcase} />

### Icon variant

Set `type="icon"` to render an SVG icon with a semantic background. The `appearance` prop controls the background color (`success`, `muted`, or `error`), and the `icon` prop accepts an icon component. The icon size is handled internally by the component.

## Accessibility

- For the image variant, provide a meaningful `alt` prop so the indicator image is announced by screen readers.
- The child element should carry its own accessibility label independently.
