# 📋 Select

<CustomTabs>

## Introduction

The Select component provides a dropdown interface for choosing from a list of options. Built on Base UI's [Combobox](https://base-ui.com/docs/combobox), it offers excellent accessibility, keyboard navigation, and a modern floating label design that matches our BaseInput component.

> View in [Figma](https://www.figma.com/design/JxaLVMTWirCpU0rsbZ30k7/2.-Components-Library?node-id=6397-463&m=dev).

## Anatomy

<Canvas of={SelectStories.Base} />

- **Trigger**: The clickable element that opens the dropdown with floating label support. Also supports [custom triggers](#custom-trigger) for button-style or fully custom rendering.
- **Label**: Optional floating label that animates when a value is selected
- **Value**: The currently selected value
- **Icon**: Chevron down/up indicator
- **Content**: The dropdown container with options. Its minimum width matches the trigger width, so the dropdown is never narrower than the trigger.
- **Search** *(optional)*: A search field at the top of the dropdown that filters items by label. Uses `SearchInput` internally.
- **Items**: Individual selectable options
- **Groups**: Optional grouping of related items with labels
- **Separators**: Visual dividers between groups

## Custom Trigger

By default, `SelectTrigger` renders an input-style trigger with a floating label. When you need a different look, pass a `render` prop to replace the default trigger entirely.

The `render` function receives two props:
- **`selectedValue`**: the currently selected value (`null` if nothing is selected)
- **`selectedContent`**: a ReactNode that renders the selected item's text

### Trigger Variants

`SelectTriggerButton` is a built-in preset that renders a compact button-style trigger. It shows a label when nothing is selected, and the selected item's text once a value is picked. It supports icons, crypto icons, all `TriggerButton` appearances, and you can also build a fully custom trigger from scratch using the `render` function directly.

## Overview

This component is built on top of [Base UI](https://base-ui.com/), which provides a robust foundation using unstyled, accessible primitives. We've customized it to match our design system while maintaining all the accessibility and functionality features.

```bash
npm install @ledgerhq/lumen-ui-react @base-ui/react
```

> **Note**: `@ledgerhq/lumen-design-core` and other peer dependencies (`react`, `tailwindcss`, `clsx`, etc.) are required. See our [Setup Guide →](?path=/docs/getting-started-setup-tailwind--docs#1-tailwind-config) for complete installation and Tailwind configuration.

The `@base-ui/react` dependency provides accessible, unstyled select functionality with proper keyboard navigation, focus management, and screen reader support.

## Basic Usage

> **Content width:** `SelectContent` has a minimum width equal to the trigger width (`min-w-[trigger-width]`). The dropdown will grow wider if items need more space but will never be narrower than the trigger.

```tsx
import {
  Select,
  SelectContent,
  SelectItem,
  SelectItemText,
  SelectList,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';

const options = [
  { value: 'option1', label: 'Option 1' },
  { value: 'option2', label: 'Option 2' },
  { value: 'option3', label: 'Option 3' },
];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select items={options} value={value} onValueChange={setValue}>
      <SelectTrigger label='Label' />
      <SelectContent>
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
            </SelectItem>
          )}
        />
      </SelectContent>
    </Select>
  );
}
```

### With Descriptions

];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select items={planOptions} value={value} onValueChange={setValue}>

      <SelectContent>

### With Groups

Add a `group` field to your items. The component groups them automatically, rendering headers, separators, and per-group collection iteration. The item render function is identical to a flat list.

```tsx

  SelectItem,
  SelectItemText,
  SelectList,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';

const produceItems = [

  { value: 'banana', label: 'Banana', group: 'Fruits' },
  { value: 'carrot', label: 'Carrot', group: 'Vegetables' },
  { value: 'broccoli', label: 'Broccoli', group: 'Vegetables' },
];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select items={produceItems} value={value} onValueChange={setValue}>
      <SelectTrigger label='Category' />
      <SelectContent>
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
            </SelectItem>
          )}
        />
      </SelectContent>
    </Select>
  );
}
```

### With Search

Drop `SelectSearch` inside `SelectContent`, above `SelectList`. The `items` prop on `Select` must still list every option; search filters the list automatically using a case-insensitive label match. Override via the root `filter` prop when you need custom matching, or pass `filter={null}` to disable filtering entirely (useful when filtering is handled externally via `filteredItems`).

```tsx
import {
  Select,
  SelectContent,
  SelectItem,
  SelectItemText,
  SelectList,
  SelectSearch,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';

const countryOptions = [
  { value: 'us', label: 'United States' },
  { value: 'ca', label: 'Canada' },
  { value: 'mx', label: 'Mexico' },
];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select items={countryOptions} value={value} onValueChange={setValue}>
      <SelectTrigger label='Country' />
      <SelectContent>
        <SelectSearch placeholder='Search countries' />
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
            </SelectItem>
          )}
        />
      </SelectContent>
    </Select>
  );
}
```

### Custom Filter

The default search matches items by `label` only. When your items carry additional searchable fields in `meta` (e.g. ticker symbols, codes, descriptions), provide a custom `filter` function to `Select`. The function receives each item and the current query, and returns `true` to keep the item visible.

```tsx
import {
  Select,
  SelectContent,
  SelectEmptyState,
  SelectItem,
  SelectItemText,
  SelectList,
  SelectSearch,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';

const tokenOptions = [
  { value: 'btc', label: 'Bitcoin', meta: { ticker: 'BTC' } },
  { value: 'eth', label: 'Ethereum', meta: { ticker: 'ETH' } },
  { value: 'sol', label: 'Solana', meta: { ticker: 'SOL' } },
];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select
      items={tokenOptions}
      value={value}
      onValueChange={setValue}
      filter={(item, query) => {
        const q = query.toLowerCase();
        return (
          item.label.toLowerCase().includes(q) ||
          ((item.meta?.ticker as string) ?? '').toLowerCase().includes(q)
        );
      }}
    >
      <SelectTrigger label='Token' />
      <SelectContent>
        <SelectSearch placeholder='Search by name or ticker' />
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
              <span className='body-3 text-muted'>{item.meta?.ticker as string}</span>
            </SelectItem>
          )}
        />
        <SelectEmptyState>No tokens found</SelectEmptyState>
      </SelectContent>
    </Select>
  );
}
```

### Async Search

When items are loaded from a remote source, use `filteredItems` and `onInputValueChange` to control filtering externally. Pass `filteredItems` with the server results — when provided, the component skips internal filtering and uses your array directly. Listen to `onInputValueChange` to trigger API calls.

```tsx
import {
  Select,
  SelectContent,
  SelectEmptyState,
  SelectItem,
  SelectItemText,
  SelectList,
  SelectSearch,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';
import type { SelectItemData } from '@ledgerhq/lumen-ui-react';

function AsyncSelect() {
  const [value, setValue] = useState<string | null>(null);
  const [filteredItems, setFilteredItems] = useState<SelectItemData[]>([]);
  const [loading, setLoading] = useState(false);

  const handleInputValueChange = async (query: string) => {
    setLoading(true);
    const results = await fetchUsersFromApi(query);
    setFilteredItems(results);
    setLoading(false);
  };

  return (
    <Select
      items={allUsers}
      filteredItems={filteredItems}
      onInputValueChange={handleInputValueChange}
      value={value}
      onValueChange={setValue}
    >
      <SelectTrigger label='Assign reviewer' />
      <SelectContent>
        <SelectSearch placeholder='Search users…' />
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
            </SelectItem>
          )}
        />
        <SelectEmptyState>{loading ? 'Loading…' : 'No results'}</SelectEmptyState>
      </SelectContent>
    </Select>
  );
}
```

> **Tip:** Consider debouncing `onInputValueChange` to avoid excessive API calls while the user is typing.

### Controlled vs Uncontrolled

The Select component supports both controlled and uncontrolled usage patterns. `onValueChange` receives `string | null` (`null` when the selection is cleared).

```tsx
// Controlled
const [value, setValue] = useState<string | null>(null);
<Select items={options} value={value} onValueChange={setValue}>

// Uncontrolled
<Select items={options} defaultValue='option1' onValueChange={(value) => console.log(value)}>
```

### Custom Trigger with `render`

The `SelectTrigger` component accepts an optional `render` prop that replaces the default input-style trigger. The function receives `{ selectedValue, selectedContent }` and must return a single root element.

#### Using `SelectTriggerButton`

`SelectTriggerButton` is a ready-made preset for button-style triggers. It accepts all `TriggerButton` props (`appearance`, `size`, `icon`, `iconType`) plus a required `label`.

```tsx
import {
  Select,
  SelectTriggerButton,
  SelectContent,
  SelectItem,
  SelectItemText,
  SelectList,
  SelectTrigger,
} from '@ledgerhq/lumen-ui-react';

const accountOptions = [
  { value: 'all', label: 'All accounts' },
  { value: 'checking', label: 'Checking' },
];

function MyComponent() {
  const [value, setValue] = useState<string | null>(null);

  return (
    <Select items={accountOptions} value={value} onValueChange={setValue}>
      <SelectTrigger
        render={(renderProps) => (
          <SelectTriggerButton
            {...renderProps}
            label='All accounts'
            icon={<Settings size={20} />}
            iconType='flat'
          />
        )}
      />
      <SelectContent>
        <SelectList
          renderItem={(item) => (
            <SelectItem key={item.value} value={item.value}>
              <SelectItemText>{item.label}</SelectItemText>
            </SelectItem>
          )}
        />
      </SelectContent>
    </Select>
  );
}
```

#### Fully custom trigger

When none of the presets fit, write your own render function. Use `selectedValue` to decide what to display and `selectedContent` to render the selected item's text.

```tsx
<SelectTrigger
  render={({ selectedValue, selectedContent }) => (
    <button className='flex items-center gap-8 rounded-sm bg-muted px-16 py-12'>
      {selectedValue ? (
        selectedContent
      ) : (
        <span className='text-muted'>Pick an option…</span>
      )}
    </button>
  )}
/>
```

> **Note:** When `render` is provided, the props `label`, `className`, and `labelClassName` have no effect — they only apply to the default trigger.
