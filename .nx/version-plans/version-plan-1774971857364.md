BREAKING_CHANGE(ui-rnative): split "editable" and "disabled" props


`editable` and `disabled` are now two separate concerns across all input components. Previously, setting `editable={false}` was the only way to disable an input and it controlled both the visual disabled styles and text editability. Now:

- **`disabled`** (new) — applies disabled visual styles and prevents all interaction.
- **`editable`** — only controls whether the user can type in the field (a "read-only" state), with no visual change.

## Affected components

- `TextInput`
- `SearchInput`
- `AddressInput`
- `AmountInput`

## How to migrate

If you were using `editable={false}` to show a **disabled** input, replace it with `disabled`:

```diff
- <TextInput editable={false} />
+ <TextInput disabled />
```

If you were using `editable={false}` for a **read-only** field (user can't type but the input should look normal), no change is needed — `editable={false}` still prevents typing and now correctly keeps the default visual appearance.
