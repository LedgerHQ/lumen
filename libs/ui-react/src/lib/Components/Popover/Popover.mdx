# 💬 Popover

<CustomTabs>
  <Tab label='Overview'>

## Introduction

The Popover component displays rich content in a floating panel anchored to a trigger element. It supports positioning, optional backdrop overlays, and both controlled and uncontrolled modes.

Built on top of [Base UI Popover](https://base-ui.com/react/components/popover) primitives, it provides accessible focus management, keyboard interactions, and automatic collision detection.

## Anatomy

- **Trigger**: The element that toggles the popover open/closed
- **Content**: The floating panel containing the popover body
- **Overlay** (optional): A backdrop that blocks outside interactions when enabled

### Width

The popover content width can be configured:

- `hug` (default): Content-fit width
- `fixed`: Fixed 400px width

You can also apply a custom width via `className` on `PopoverContent` when using `hug`.

<Canvas of={PopoverStories.WidthShowcase} />

### Position

Position the popover on any side of the trigger with the `side` prop, and align it with the `align` prop. The popover automatically repositions to avoid viewport collisions.

<Canvas of={PopoverStories.PositionShowcase} />

Available sides: `top`, `bottom` (default), `left`, `right`

Available alignments: `start` (default), `center`, `end`

### Overlay

When `overlay` is `true`, a backdrop is rendered and the popover enters modal mode, blocking scrolling and pointer events on elements behind it.

Use `createPopoverHandle` to connect triggers that live outside the `Popover` tree. Each trigger can pass a typed `payload` that the popover receives via a render function:

## Responsive Behavior

- The popover automatically repositions itself to stay within the viewport
- Content width adapts to its children in `hug` mode, or stays fixed at 400px in `fixed` mode
- Use `className` on `PopoverContent` for custom responsive widths

## Accessibility

- **Keyboard Navigation**: Escape closes the popover, Tab moves focus within the content
- **Focus Management**: Focus is trapped inside the popover in modal (overlay) mode
- **Screen Reader Support**: Proper ARIA attributes via Base UI primitives
- **Backdrop**: When `overlay` is enabled, outside elements are inert

## Installation

```bash
npm install @ledgerhq/lumen-ui-react @base-ui/react
```

> **Note**: `@ledgerhq/lumen-design-core` and other peer dependencies (`react`, `tailwindcss`, `clsx`, etc.) are required. See our [Setup Guide →](?path=/docs/getting-started-setup-tailwind--docs#1-tailwind-config) for complete installation and Tailwind configuration.

## Basic Usage

```tsx
import {
  Popover,
  PopoverTrigger,
  PopoverContent,
} from '@ledgerhq/lumen-ui-react';
import { Button } from '@ledgerhq/lumen-ui-react';

function MyComponent() {
  return (
    <Popover>
      <PopoverTrigger>
        <Button appearance='gray'>Open Popover</Button>
      </PopoverTrigger>
      <PopoverContent>
        <p>Popover content goes here.</p>
      </PopoverContent>
    </Popover>
  );
}
```

## Do's and Don'ts

  </Button>
