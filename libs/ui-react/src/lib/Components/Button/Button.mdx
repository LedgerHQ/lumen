## Introduction

Buttons are key interactive elements that trigger actions and guide users through the interface. They come in various styles and states to reflect hierarchy, intent, and context

> View in [Figma](https://www.figma.com/design/JxaLVMTWirCpU0rsbZ30k7/2.-Components-Library?node-id=9-28&m=dev).

## Anatomy

<Canvas of={ButtonStories.IconText} />

- **Label(required)**: must be present and describe the action that occurs when the button is tapped
- **Icon (optional)**: visually communicate the button's action, and helps draw attention

> Note: For **icon-only buttons**, use the [IconButton](/docs/react-iconbutton--docs) component instead. It's specifically designed for icon-only interactions with proper accessibility support.

There are five button appearances, which indicate its emphasis. From strongest to weakest:

- accent
- base (default)
- gray
- transparent

### Size

Buttons come in four different sizes:

- `sm` (**40px**)
- `md` (**48px**, default)
- `lg` (**56px**)

> Note: For `xs` size (32 px), use the [IconButton](/docs/react-iconbutton--docs) component instead.

### States

> **Important Note:** The `loading` and `disabled` states are **not coupled**. When `disabled` is set, all event handlers are deactivated. When only `loading` is set (without `disabled`), the button remains clickable. Combine both props if you need to prevent interactions during loading.

### With Icons

The Button supports icons with automatic size adjustment based on the button size.

<Source of={ButtonStories.IconText} />

### Full Width

Use the `isFull` prop to make the button take the full width of its container:

```tsx
<Button size='md' appearance='base' isFull>
  Full Width Button
</Button>
```

### Loading and Disabled States

The `loading` and `disabled` props are **independent**: `disabled` deactivates all event handlers and makes the button non-interactive, while `loading` shows a spinner but keeps the button clickable. Combine both to prevent interactions during loading.

<Source of={ButtonStories.StatesShowcase} />

### Using with Routers (asChild)

The `asChild` prop merges Button styles with a child element, ideal for routing libraries. The child handles its own content; `icon` and `loading` are applied to the child.

<Source of={ButtonStories.AsChild} />

### Custom Styling

Extend layout with Tailwind classes (layout utilities only, e.g. positioning or responsive visibility):

```tsx
<Button size='md' appearance='base' className='md:hidden'>
  Custom Button
</Button>
```
